BUILD149_UnifiedURDFPlusLoader

- Eliminado el switch standard-URDF vs URDF+ del entrypoint.
- URDF_Viewer/urdfplus_viewer_main.js ahora usa un único pipeline URDF+ para ambos casos.
- El mecanismo visual se basa en el sistema antiguo probado: AssetDB + URDFLoader + loadMeshCb.
- Esto replica la parte que sí funcionaba: resolución de package://, basename, subpaths, DAE, texturas y thumbnails.
- URDFLoader ignora tags URDF+ desconocidos sin romper el armado estándar del robot.
- Se mantiene el shell modular: ComponentsPanel, ToolsDock, selección, hover, isolate/show-all y thumbnails.
- Sin raw.githubusercontent; URDFLoader se resuelve por jsDelivr.

Motivo:
El pipeline BUILD148 hacía una distinción externa entre URDF estándar y URDF+. Este build elimina esa distinción: URDF+ es el sistema único de carga/simulación visual y acepta archivos URDF y URDF+.
