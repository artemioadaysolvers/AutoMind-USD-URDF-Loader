BUILD152 - Unified URDF+ Joint Simulation + Texture/Thumbnail Fix

- Keeps one URDF+ entrypoint for both ordinary URDF and URDF+ XML.
- Inlines URDF <texture filename="..."> references to data:image/*;base64 URLs before URDFLoader parses the XML.
  This stops Colab from requesting /meshes/*.png through its runtime proxy and fixes material loss.
- Legacy AssetDB now accepts raw base64 and data URLs, normalizes them, and inlines Collada <init_from> texture references.
- Rebuilds assetToMeshes after asynchronous URDFLoader/Collada mesh loading settles and refreshes the Components panel.
- Adds fallback grouping by ancestor link name when a mesh has no __assetKey, so component thumbnails are created even when URDFLoader does not tag meshes.
- Replaces URDF joint/loop arrows with USD-style batched LineSegments: automind_joint_axes_batch and automind_loops_batch.
- Show Joints / Show Loops now expose information the same way as USD Viewer decorations: thin line segments, depth-safe, clipped by section plane.
- Joint drag reads URDFLoader jointValue[0] and keeps it synchronized, improving movable joint simulation.
- Preserves all advanced UI from BUILD151: finite model-scaled grid, orthographic depth safety, gradual show-all to current render mode, component click camera tween, render modes, section picking, isolate/show-all, thumbnails and ToolsDock.
