AutoMind MJCF Viewer BUILD176 validation
=========================================

Purpose
-------
Repair closed-chain parity with the working USD/URDF viewers.

Implemented
-----------
1. Replaced the old one-shot MJCF loop correction with the same interaction model
   used by the USD/URDF viewers:
   - Damped least-squares projection of passive closure joints.
   - Adaptive damping and bounded per-iteration updates.
   - Active dragged joint is pinned; equality/joint slaves remain exact.
   - Infeasible mouse drags use a binary search and stop at the farthest closed pose.

2. Fixed MJCF equality frame semantics:
   - equality/connect anchor is local to MJCF body1, not world-space.
   - equality/weld uses the supplied relpose/anchor frames when present.
   - default welds derive their frames from the imported reference pose.
   - omitted body2 is represented by a stable virtual world body.
   - weld rotational closure respects torquescale, including torquescale=0.

3. Preserved all BUILD175 texture repairs.

Static verification performed
-----------------------------
- node --check MJCF_Viewer/core/MJCFCore.js : PASS
- Confirmed previous global-anchor conversion no longer occurs.
- Confirmed closed-loop rendering and DLS share localPos/localFrame state.

Limitation
----------
The robot MJCF/OBJ/PNG source used in the screenshot was not inside the supplied
viewer source ZIP. The source has therefore been syntax-validated and aligned to
the MuJoCo frame semantics, but the exact gripper was not executable locally.
